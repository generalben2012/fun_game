# 수리 마법사 (가제) - Godot PC 액션 게임 개발 태스크 리스트

본 문서는 기획서(story.md, feature.md, app_flow.md) 기반으로 Godot Engine(PC)에서 게임을 개발하기 위한 상세 작업(Task) 리스트입니다.

## Phase 1. Godot 개발 환경 구축 및 기본 뼈대
- [x] 1. Godot 4.x 프로젝트 생성 및 PC(Windows/Mac) 해상도 세팅.
- [x] 2. 기본 디렉토리 구조(Scenes, Scripts, Resources, Assets) 정리.
- [x] 3. 플레이어 캐릭터(이수식) Scene 생성 및 WASD 8방향 이동 스크립트(CharacterBody2D) 구현.
- [x] 4. 메인 뷰포트(카메라) 세팅 및 카툰 렌더링 스타일의 기본 타일맵 테스트 환경 구성.

## Phase 2. 코어 시스템 전반: 수리 영창 모듈
- [/] 5. 전투 중 [공격] 키 입력 시 게임 일시 정지(Engine.time_scale 제어) 로직 구현.
- [/] 6. 시간 정지 시 화면 중앙에 내려오는 '칠판 UI (수리 영창 패널)' Scene 제작.
- [/] 7. **난수 생성 및 문제 출제 시스템 구현**:
  - 더하기, 빼기, 구구단 등 기초 연산 난수 생성기.
- [ ] 8. **문제 입력 시스템 구현**:
  - 주관식 모드: 키보드 입력을 받아 String으로 검증하는 UI.
  - 객관식 모드: 4지선다 버튼(마우스 클릭) UI 로직 구현.
- [ ] 9. 시간 정지 상태에서 실시간으로 깎이는 '뇌용량(마나)' 게이지 바 UI 구현 (0이 되면 캐스팅 강제 취소).

## Phase 3. 전투 액션 및 스킬 연출 (챕터 1 기준)
- [ ] 10. 연습용 에너미 AI (플레이어를 향해 다가오는 기본 몹) Scene 생성.
- [ ] 11. 정답 처리 시 플레이어 좌표에서 몹을 향해 발사되는 마법 투사체(Projectile) 생성.
  - 마법 1: 덧셈/뺄셈 수리검 (직선 관통형 데미지).
  - 마법 2: 구구단 텍스트 기관총 (빠른 연사형 연출).
- [ ] 12. 에너미 피격 판정(Hitbox) 및 넉백, 사망 연출.
- [ ] 13. 오답 시 칠판 UI가 흔들리며 '분필 부러지는 소리'와 함께 취소되는 페널티 연출.
- [ ] 14. 몹 드롭 아이템(이과의 혼, 당분 포션) 생성 및 플레이어 획득(자석 효과) 로직 구현.

## Phase 4. 게임 플로우 상태 관리 및 UI/UX
- [ ] 15. 스플래시 화면 및 타이틀 메뉴 씬(Title Scene) 제작. (Start, Exit 등)
- [ ] 16. 스테이지 선택 화면(World Map Scene) UI 구성 및 Scene 매니저를 통한 씬 전환(Load) 구현.
- [ ] 17. 전투 중 ESC 키 일시정지 팝업 메뉴 구현.
- [ ] 18. 스테이지 클리어/실패 시 띄워주는 Result Screen UI 제작.

## Phase 5. 고도화 및 폴리싱 (이후 확장)
- [ ] 19. 챕터 2, 3의 고급 스킬(이차함수 포물선, 인수분해, 역함수 무지개 반사 등) 물리 엔진 적용 및 연출 추가.
- [ ] 20. 보스 몬스터 패턴(1, 2, 3 페이즈 분기) State Machine 구현.
- [ ] 21. 병맛/카툰 렌더링 에셋(이미지, BGM, 효과음) 로드 및 일괄 교체.
